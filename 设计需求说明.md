eCas
=====

    数据缓存与存储
    之前完善了eMysql/ePgdb数据库相关的操作 现在完全可以基于eMysql/ePgdb封装一套数据库缓存与存储的整套方案 方便在游戏项目中使用 这样应用层以后就不用操作数据库的加载和落地操作 完全只用开始定义好schema之后 在应用中使用进程内的数据或者缓存中的数据 就很简单便捷了

Build
-----

    $ rebar3 compile

# 设计要点

    模块和变量 宏定义 等等相关采用驼峰命名法 
    
    数据缓存使用ets表做换成  落地可以采用mysql 或者 postgresql 已经完成了对db的封装 在项目依赖中已经配置eMysql/ePgdb驱动启动项目的时候可以传入参数模块(eMysql/ePgdb)指定落地数据库
    schema 定义引用eMysql/ePgdb的scema 定义格式 但是-record(schema, {repr = map,          %% record | map - Erlang 端数据表示方式comment = "",      %% string()|binary() - 表注释 fields = []          %% [#schField{}]}).
    这个结构得加个字段 cache = undefined(这个表不会建缓存) | #{type = whole | hotData(整表缓存|部分热数据缓存), saveInterval = 5*60(存储间隔), saveType = whole | chageData(保存的时候直接存整条数据还是查看修改的字段做修改的字段存储),  loadFun = undefined(走一个通用加载函数即可) | {M, F, A}(初始化加载函数)}
    
    需要将这个schema 导出为db使用的 类似于eMysql/ePgdb的 genDbSchema 还得将上面cache 相关的信息导出来 要导出哪些表需要换成 以及一些其他缓存信息 

    缓存相关的表一个表一个进程 用来定时落盘 
    缓存相关的表一个表一个ets 其中如果不是全缓存还得建一个keys的ets表 将数据库的所有的key 加加载到这个keysets表里面

    如果不是整条缓存 通过ePtDirty这个项目来实现脏字段的收集 
    所有缓存表的操作 都要通过eGLock加锁操作
    
    另外就是删除数据库数据的时候 从缓存表删除的时候 还得删除keys表(如果有) 还是即使删除数据库里面的数据 然后就是新增表的时候 又如何保证第一次落盘插入整条数据呢 
    同时修改多个ets表的数据可以参考 eGLock:transaction 函数  如果操作缓存表达时候 表是全缓存表的话 就直接用缓存表读 如果没有数据就是没有 不用再去数据库读取 如果不是全缓存的表就先去缓存表读取 如果没读取到则去keys表查看 如果有key再去数据库读取数据 如果没有则表示数据库都没有这个数据
